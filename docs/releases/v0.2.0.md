# 0.2.0

## Summary

- Strengthened the hosted demo into a steadier reviewer-ready release.
- Added deployment metadata, hosted smoke automation, and clearer maintainer/reviewer runbooks.
- Polished the local Streamlit reviewer UI so responses are easier to scan and the interface is less distracting.

## Highlights

### Hosted deployment and trust signals

- Added deployed build metadata and surfaced it through `/version`.
- Added `X-Request-Id` headers on core API responses for easier traceability.
- Added GitHub Actions hosted smoke automation for live verification of the deployed demo.
- Improved hosted smoke reporting with step summaries and downloadable artifacts.

### Reviewer and maintainer workflow

- Expanded reviewer-facing release assets, curl examples, and review-kit docs.
- Added a maintainer runbook for password rotation, secret updates, redeploys, smoke validation, and troubleshooting.
- Improved the hosted landing page, release checklist, and deployment documentation for smoother handoff and sharing.

### Streamlit reviewer UI polish

- Renamed the approvals UI to Incident Escalation for clearer business framing.
- Added visible in-progress query feedback so users can see when the copilot is working.
- Reformatted long answer blocks into more scannable structured output.
- Deduplicated citation display by document.
- Removed the extra query-response header and hid Streamlit deploy controls from the reviewer UI.
